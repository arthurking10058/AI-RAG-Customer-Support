# 展示与截图指南

## 截图目标

- 让别人一眼看懂这是一个 `Agent + 混合检索 RAG` 项目。
- 让截图能直接放进 README、简历附件和面试讲解稿。

## 建议截图

### 1. 首页总览

- 画面：`app.py` 首页
- 要点：标题、模式切换、用户上下文、推荐问题
- 目的：说明项目是可演示的作品，而不是纯代码仓库

### 2. 普通知识问答

- 问题：`深圳最近比较潮湿，扫拖一体机器人要怎么保养？`
- 要点：回答中带参考来源
- 目的：体现 `Agent + RAG` 的知识问答能力

### 3. 月度报告

- 问题：`给我生成这个月的使用报告`
- 要点：固定 `Markdown` 结构、外部记录、保养建议
- 目的：体现报告模式和工具调用能力

### 4. 结构图

- 内容：项目目录或调用链
- 目的：帮助面试官快速理解整体架构

## 截图顺序

1. 先截首页
2. 再截普通问答
3. 再截报告结果
4. 最后补一张结构图

## 备注

- 截图优先使用当前仓库已有的稳定版本。
- 不建议为了截图临时改成更复杂的企业级架构。
